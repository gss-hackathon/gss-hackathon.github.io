呼叫 API (Raw)

1.API介紹

在平臺中可以透過呼叫API Raw或者呼叫API key value兩種節點來進行API的串接，這兩個節點主要差別在於Body的形式，在平臺使用API RAW以及API key and value的節點，就如同使用Postman對API發送請求，接著填上必要資訊如:URL等，進行模擬測試即可得知是否正常。

請求類型如下：

  • Get
  • Post
  • Put
  • Patch
  • Delete
  • Options

2.API(Raw)操作

(1)新增呼叫API (Raw)節點

在腳本中新增呼叫API (Raw)節點，修改名稱，輸入請求類型、網址等必要資訊，進入API模擬測試，試著發送看是否成功，成功會出現200OK。

(2)設定API (Raw)節點變數

到變數中新增一新的變數，資料類型選擇其他變數的值，目標變數選擇自訂輸入自定義名稱，設定自訂變數值選擇節點輸出類的呼叫API(Raw)中的回應的Json物件，最後按下儲存

(3)測試API結果

新增發送訊息節點，修改名稱，在訊息類選擇剛剛定義的變數，儲存後到模擬測試檢查是否正確即完成。
