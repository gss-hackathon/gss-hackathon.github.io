調適型卡片

1.Microsoft Adaptive Cards

Adaptive Cards是一種開放式卡片交換格式(JSON格式)，主要分成卡片主體及卡片資料兩個部份。可以自動適應不同應用程式之間的卡片風格轉換。例： Microsoft Teams、 Bot Framework…等。因此開發人員只需設計一套卡片即可在不同的應用程式之間通用。

相關參考網址:

  • Document：Adaptive Cards官方說明文件。
  • Schema Explorer：Adaptive Cards元件說明、適用版本及使用方式說明。
  • Samples and Templates：Adaptive Cards設計範本。
  • Designer：Adaptive Cards線上編輯器。

我們將使用Adaptive Cards線上編輯器(Designer)快速並有效率進行卡片的設計。

網址：https://adaptivecards.io/designer/

2.Adaptive Cards Designer 線上編輯器介紹

  • 上方功能列：可以挑選host app、Undo、Redo、Copy card payload、Preview mode（預覽卡片)、Target version（版本挑選)等。

• 左側Card Elements：您可以用拖拉方式將元件放置於中間的卡片編輯區（相關的元件介紹請參閱下方)。

• 中間卡片編輯區及Card Structure，點選卡片上的元件，Card Structure會自動選取該元件並在右側顯示該元件的相關屬性。

• 右側Element Properties：可以設定元件的屬性。

• 下方Card Payload Editor以及Sample Data Editor：

Card Payload Editor：顯示此張卡片的JSON，若您已相關熟悉Adaptive Cards，也可以直接編修JSON內容。
Sample Data Editor：此張卡片的資料來源，卡片欲使用資料來源的資料，語法請參照Binding to data說明文件。

3.基本元件介紹

Adaptive Cards提供相當豐富的元件，可根據您的需求選用不同的元件來設計您的卡片，每一種元件均有各自的屬性可以設定。 在使用元件時，請務必參考各元件的屬性說明及注意使用版本。

C.ai BotBuilder 目前僅可支援1.4以下版本

(1)基本類型元件

如果您想要單純顯示文字以及圖片或多媒體可以使用以下的元件。

  • TextBlock：顯示文字，並可設定文字的Size、Color、Weight等屬性。
  • RichTextBlock：可以將文字內容拆成不同的文字區段，各自設定顯示的格式。
  • Image：顯示圖片。
  • Media：顯示多媒體。

(2)容器類型元件

容器類型的元件可以使您更方便的編排元件的顯示位置，以及群組化的控制顯示或隱藏等設定。

  • Containers：元件容器，可以在此容器內再嵌入其他的元件，例如Containers、TextBlock、ColumnSet…等。
  • ImageSet：圖片元件容器，此容器內允許您加入多張的圖片，並依序顥示這些圖片。
  • FactSet：資料顯示容器，需要提供資料來源，並設定資料的標題及內容值，條列式顥示資料。
  • ColumnSet：ColumnSet 將區域劃分為 Columns，允許元件並排放置。
  • Table：提供一種以表格形式顯示資料的方法。(須支援1.5版才能使用)
  • ActionSet：按鈕的集合。

(3)輸入類型元件

輸入類型的元件，可以讓使用者輸入資料，像是輸入文字、數字、日期與時間。或是使用下拉式選單讓使用者挑選所需項目。

* 注意:輸入類型的元件，務必設定元件的Id屬性。如果該元件設定為必填(Required)，也須一併設定錯誤訊息(Error message)。當使用者未輸入必填欄位時，才能提示使用者錯誤訊息。

  • Input.Text：文字輸入元件。
  • Input.Date：日期輸入選擇元件。
  • Input.Time：時間輸入選擇元件。
  • Input.Number：數字輸入元件。
  • Input.ChoiceSet：下拉式選擇元件，可以設定Style屬性展開或是下拉方式讓使用者選擇項目。
  • Input.Toggle：讓使用者在兩個項目之間單一選擇。例如勾選或不勾選。

(4)按鈕類型元件

可依照您所想要的按鈕動作挑選以下幾種的按鈕類型。

  • Action.OpenUrl：點選按鈕後開啟連結。
  • Action.Submit：點選按鈕後送出資料。
  • Action.ShowCard：點選按鈕後顯示內嵌的卡片。
  • Action.ToggleVisibility：點選按鈕後控制其他元件顯示或隱藏。
  • Action.Execute：收集表單上的輸入元件資料，並可與您所設定的資料合併後再向Client發送事件。

4.如何使用Adaptive Cards

我們使用Adaptive Cards線上編輯器(Designer)完成卡片的設計後，只要將Card Payload Editor的JSON內容先整個拷貝，回到 C.ai BotBuilder的腳本中，使用「發送訊息」或「文字輸入」節點，文字類型請挑選：調勢型卡片，並將拷貝的JSON內容貼進"調適型卡片 Json"輸入框中即可。

如若在設計時有搭配資料來源（Sample Data Editor中的資料），必須要在進入節點前完成資料來源的變數設定，並指定使用該變數，才能正確的顯示卡片。

5.模擬測試

使用模擬測試執行到節點時，可以讓您即時觀看所設計的卡片顯示是否正確。
