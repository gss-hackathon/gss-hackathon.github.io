變數管理

1.自訂變數

在機器人管理的腳本中新增節點。

在節點中的變數中新增。

選擇資料型態，目標變數選擇自訂，選擇全域或腳本變數(全域變數：在同一機器人中所有腳本都可使用 ；腳本變數：在同一腳本中才可使用)，輸入自訂義名稱按下打勾。

設定自訂變數值選擇所需的值。

接著在其他節點中使用即完成。

2.對話類訊息

在機器人管理的腳本中新增節點。

使用對話類訊息，依需求取用所需要的變數，按下儲存即完成。

3.全域參數

在機器人管理中的進階選擇全域參數，點選新增。

輸入顯示名稱、名稱及模擬環境的值，按下儲存。

接著到腳本中選取全域參數做使用即完成。

4.訊息類輸出

訊息類輸出是在變數選擇的「訊息類」裡面，只要是在對話框裡面的變數都可以在「訊息類」找到，根據使用情境選擇。

(1) 取得訊息

取得使用者的訊息資料重複發送一次。

(2) 取得訊息卡片資料

常用於 Adaptive Card、Line Flex 或是表單卡片中收集使用者輸入的內容。

此範例使用「表單卡片」，在欄位設定好 ID，點擊「送出」後下一個節點「發送訊息」展示出「訊息卡片資料」的內容。

(3) 附件訊息

取得使用者上傳附件的基本資料。

  • 要使用上傳附件功能需要先到「發佈設定」勾選「上傳附件」。
  • 使用者的「附件訊息」為一組 JSON 格式，資料量比較大建議使用「偵錯」節點查看。
    • 附件訊息 JSON 簡易說明：
      • contentType：檔案類型
      • contentUrl：附件上傳位置
      • name：檔案名稱
      • thumbnailUrl：Base64 的縮圖網址

5.節點輸出

節點輸出是在變數選擇的「上個節點輸出類」，主要是方便選取上一個節點輸出內容的方法，常用於不想將上一個節點內容存成變數而是直接使用。

(1)選擇上個節點輸出

  • 這裡使用多種節點串接，並直接選取上個節點的輸出。
  • 需要注意當直接使用上個節點來獲取資料，中間突然加入其他節點會造成無法正常取得資料。
