Cai_ETA 訊息推播_API_v2

1.ETA 訊息發送 API

(1)發送文字訊息

適用於單純發送⽂字訊息

  • URL
POST https://{hostUrl}/eta/api/subscription/{botId}/message/multicast

  • hostUrl – 請填入 ETA server (通常為 bot server) 的網址
  • BotId – 請填入 Bot Id (Bot 會有一組固定的識別碼)

  • Header
x-gss-event-subscription-key: {subscription key}
x-gss-event-from: {botId}
content-type: application/json
參數名稱 說明 格式 必填
x-gss-event-subscription-key Bot 訂閱 Key，Bot 安裝後的首次發佈時會產生一組 subscription key。 string V
x-gss-event-from 發送來源，填寫系統名稱或是 botId；可留空白 string
content-type 內容資料型態，固定值 : application/json string V
  • Body
參數名稱 說明 格式 必填
TriggerId 預設填入 botId，若有設定其他 TriggerId 會另行提供 string V
Conversations 訊息接收者 array V
Id 頻道為 Iota/ Teams/ Line : 留空白 其他頻道如 Webchat : 使用者的聊天室 Id 若 Webchat 有介接登入認證：可留空白 string V
RecipientId 訊息接收者的識別碼，如： • iota: 使用者帳號, 如: 員工編號或 AD 帳號 • Teams: UserId，為亂數字串 • Line: 使用者綁定的 line Id，為亂數字串 • Webchat 若有介接登入認證，則傳使用者帳號 string V
ChannelList 只推播到特定 Channel；若未限定則會發送到所有在發佈設定所勾選的頻道。 string
Subject 標題，僅作為內部紀錄查詢⽤ string
IsGroup 是否為群組 個人聊天室 : false 群組聊天室 : true boolean V
Message 文字訊息 json V
ItemType 訊息內容類型 ，請填 text string V
Item 訊息內容 string V
TemplateId 預留欄位，請留空白 string V
Data 訊息內容所需的資料，⽤於動態綁入訊息內容 json
Event 事件訊息，請填 null object V 
{
    "TriggerId": "gssbot",
    "Conversations": [
        {
            "Id": "", //請依據發佈頻道填寫
            "RecipientId": "user1",
            "ChannelList": {
                "InclusionChannels": "iota"
            }
    "Subject": "",
            "IsGroup": false
        },
        {
            "Id": "", //請依據發佈頻道填寫
            "RecipientId": "user2",
            "Subject": "",
            "IsGroup": false
        }
    ],
    "Event": null,
    "Message": {
        "TemplateId": "",
        "ItemType": "text",
        "Item": {
            "Text": "Hello {{$.User}}. Today is {{=> DateTime.Today.ToString(DateFormat);}}"
        },
        "Data": {
            "User": "Marty",
            "DateFormat": "yyyy-MM-dd"
        }
    }
}
  • Response
參數名稱 說明 格式 必填 參數名稱
status 訊息回應碼 string V 200 表⽰成功 401 表⽰認證失敗 500 表⽰其他錯誤
message 處理結果 string V 非 200 會告知錯誤原因 
{
    "status": 200,
    "message": "OK",
    "data": null
}

(2)發送事件訊息

適⽤於觸發流程或複雜內容

  • URL
POST https://{hostUrl}/eta/api/subscription/{botId}/event/multicast

  • hostUrl – 請填入 ETA server (通常為 bot server) 的網址
  • BotId – 請填入 Bot Id (Bot 會有一組固定的識別碼)

  • Header
x-gss-event-subscription-key: {subscription key} 
x-gss-event-from: {botId}
content-type: application/json
參數名稱 說明 格式 必填
x-gss-event- subscription-key Bot 訂閱 Key，Bot 安裝後的首次發佈時會產生一組 subscription key string V
x-gss-event-from 發送來源(填寫系統名稱或是 botId)，可留空白 string
content-type 內容資料型態，固定值 : application/json string V
  • Body
參數名稱 說明 格式 必填
TriggerId 預設填入 botId，若有設定其他 TriggerId 會另行提供 string V
Conversations 訊息接收者 array V
Id 頻道為 Iota/ Teams/ Line : 可留空白 其他頻道如 Webchat : 使用者的聊天室 Id 若 Webchat 有介接登入認證，則可留空白 string V
RecipientId 訊息接收者的識別碼，如： • iota: 使用者帳號, 如: 員工編號或 AD 帳號 • Teams: UserId，為亂數字串 • Line: 使用者綁定的 line ID，為亂數字串 • Webchat 若有介接登入認證，則傳使用者帳號 string V
ChannelList 只推播到特定 Channel；若未限定則會發送到所有在發佈設定所勾選的頻道。 string
Subject 標題，僅作為內部紀錄查詢⽤ string
IsGroup 是否為群組 個人聊天室 : false 群組聊天室 : true boolean V
Event 事件訊息 json V
Name 事件名稱，設定欲觸發的事件名稱。 值需配合 bot 設定 (由 bot 提供參數值) string V
Value 流程需帶的變數資料 json V
Message 文字訊息，請填 null json V 
{
    "TriggerId": "gssbot",
    "Conversations": [
        {
            "Id": "", //請依據發佈頻道填寫
            "RecipientId": "User1",
            "ChannelList": {
                "InclusionChannels": "iota"
            }
    "Subject": "",
            "IsGroup": false
        },
        {
            "Id": "", //請依據發佈頻道填寫
            "RecipientId": "User2",
            "Subject": "",
            "IsGroup": false
        }
    ],
    "Event": {
        "Name": "approval",
        "Value": {
            "FormID": "0001"
        }
    },
    "Message": null
}
  • Response
參數名稱 說明 格式 必填 其他說明
status 訊息回應碼 string V 200 表⽰成功 401 表⽰認證失敗 500 表⽰其他錯誤
message 處理結果 string V 非 200 會告知錯誤原因 
{
    "status": 200,
    "message": "OK",
    "data": null
}

2.補充說明

(1) botId、subscription key：可登入管理平台，由 機器人基本資訊取得識別碼 (botId) 及通知金鑰 (subscription key)
